## Context

当前任务详情页虽然可查看状态和 JSON，但缺少面向投资决策的“快速读数”能力。用户需要在同一页面直接判断组合在区间内的收益与风险表现，并且字段显示应统一中文，不要求用户理解底层英文字段名。

## Goals / Non-Goals

**Goals:**
- 为已完成任务提供核心指标卡片与趋势图，减少阅读原始 JSON 的负担。
- 保留原始明细视图，同时输出中文字段版本，兼顾可读性与可审计性。
- 支持新增字段的中文兜底映射，避免每次加字段都必须先改前端页面。

**Non-Goals:**
- 不引入第三方图表库，优先使用原生 SVG 维持依赖最小化。
- 不改变回测计算引擎结果口径，仅做展示层转换与派生（如回撤序列）。
- 不实现多页面报表导出。

## Decisions

### 1) 结果图表采用轻量 SVG 实现
- 决策：在详情页内通过原生 SVG 绘制折线图（净值、回撤）。
- 原因：当前曲线需求简单，避免额外依赖与体积增长。

### 2) 回撤曲线在前端按净值序列派生
- 决策：以前端 `equity_curve` 为输入，按“当前净值相对历史峰值跌幅”计算回撤。
- 原因：后端无需新增字段即可得到可读风险轨迹，并与已有指标口径一致。

### 3) 字段中文化采用“显式映射 + token 兜底”
- 决策：已知字段使用固定映射，未知字段走 token 分词映射与可读化处理。
- 原因：兼顾准确性与扩展性，避免后续新增字段出现全英文回退。

### 4) 结果页保留中文 JSON 明细
- 决策：在卡片/图表之外保留本地化后的指标与净值样本 JSON。
- 原因：支持快速核对和调试，且不牺牲透明度。

## Risks / Trade-offs

- [前端计算回撤与后端指标存在小数误差] -> 使用统一公式并限制展示精度，强调展示用途。
- [未知字段 token 兜底不够语义化] -> 优先保障可读，后续可按真实字段补充显式映射。
- [SVG 图在极长序列上可读性下降] -> 当前保留基础线图，后续按需要增加抽样和缩放。
